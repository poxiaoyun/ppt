# Slide 07: 异构算力统一调度与管理

## 页面逻辑与数据映射
- **主题**：说明 Rune AI 如何把异构芯片接入、资源池化切分和智能调度整合成一套统一底座。
- **核心观点**：打破硬件壁垒，最大化算力投资回报。
- **信息映射**：
  - 顶部标题区：明确“统一调度与管理”的主题
  - 核心观点条：一句话收束整页
  - 中部三段流程：`多厂商支持 -> 资源池化与切分 -> 智能调度`
  - 底部结论条：强调企业级统一算力底座

## 布局体系 (Three-Stage Pipeline Layout)
采用 **顶部标题 + 核心观点条 + 三段式主流程 + 底部结论条**：
1. **顶部标题区**
   - 标题左对齐，延续前面章节页的统一标题语言。
   - 副标题说明异构算力统一治理带来的价值。
2. **核心观点条**
   - 用一条轻量信息条先讲清“最大化投资回报”的核心价值。
3. **三段式主流程**
   - 第一段：多厂商支持，承接 NVIDIA、昇腾、海光、阿里云等异构算力兼容。
   - 第二段：资源池化与切分，承接共享、切分、vNPU 模板等能力。
   - 第三段：智能调度，承接 binpack、DRF、Gang 等调度策略。
   - 三段之间用箭头连接，强调统一链路而非孤立功能点。
4. **底部结论条**
   - 用一句话收束从“接入 -> 池化 -> 调度”的完整闭环。

## 视觉风格
- **背景**：延续白底科技感，并加入轨道、网格和中心光晕增强“底座调度”氛围。
- **主体表现**：
  - 三张主卡采用统一冷色系，但各自略有差异，体现阶段分工。
  - 卡片内部使用标签、指标卡与策略胶囊增强结构感。
- **品牌映射**：
  - 蓝色：多厂商接入与统一基础设施
  - 青绿色：资源池化与利用率提升
  - 蓝灰色：调度策略、秩序与治理

## 入场动画序列
依赖 `.active` 类触发：
1. `0.12s`：标题区淡入。
2. `0.26s`：核心观点条上浮出现。
3. `0.42s / 0.56s / 0.70s`：三张主卡依次显现。
4. `0.48s`：中间箭头出现，连接三段流程。
5. `0.90s`：底部结论条淡入。

## 交互设计
- 三张主卡在 `hover` 时轻微抬升，增强阶段模块感。
- 箭头仅承担流程指引作用，不做复杂动画，避免抢夺主信息。
- 标签与指标卡帮助观众快速扫读异构算力治理的关键能力。
