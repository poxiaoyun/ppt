# Slide 08: 一站式模型推理与微调服务

## 页面逻辑与数据映射
- **主题**：展示 Rune AI 如何把开发、微调、运行、发布与观测整合成一条可直接交付生产的模型服务链路。
- **信息映射**：
  - 顶部标题区：明确“推理与微调一体化”的主题
  - 左侧能力列：`开发 / 微调 / 虚拟化 / 弹性推理 / 全方位观测`
  - 右侧展示区：3 x 2 仿真界面矩阵，分别对应 `开发环境 / 推理服务 / 虚拟化桌面 / 实验数据 / 运行 Metrics 监控 / 持久化数据管理`
  - 底部结论条：强调交付标准化

## 布局体系 (Capability Rail + Image Grid)
采用 **顶部标题 + 左侧五张能力卡 + 右侧 3x2 图片展示区 + 底部结论条**：
1. **顶部标题区**
   - 标题左对齐，延续章节页统一风格。
   - 副标题概括从研发到上线的完整链路。
2. **左侧五张能力卡**
   - 每张卡仅承载一个能力点，信息密度均匀。
   - 用编号与英文副标题补充结构感。
3. **右侧图片展示区**
   - 采用 3 列 2 行仿真页面矩阵。
   - 不再使用纯占位框，而是参考 Slide 10 的 `Moha Hub Console` 语言，为每个能力点绘制一个缩微产品界面。
   - 六张仿真界面分别承接：
     - `开发环境`：仿真 VSCode IDE 缩略页，并体现 SSH 登录状态
     - `推理服务`：强调 `DeepSeek-V3.2` 模型、GPU 卡数、整体请求和负载状态
     - `虚拟化桌面`：强调放大打开的 Windows 桌面窗口
     - `实验数据`：内置学习率、loss、epoch 等走势折线
     - `运行 Metrics 监控`：关注 GPU / CPU / 内存 / 温度等变化
     - `持久化数据管理`：以 filebrowser 形态展示目录与挂载状态
4. **底部结论条**
   - 用一句话收束“开发、微调、运行、发布、观测协同”的最终价值。

## 视觉风格
- **背景**：白底科技感，延续浅网格和中心光晕。
- **主体表现**：
  - 左侧为冷色系能力卡，右侧为整块展示面板。
  - 右侧每个小卡都是“微型控制台”，延续 Slide 10 的截图式产品界面语言，但简化到单卡尺度。
  - 当前版本已移除每张小卡顶部的文字标签，只保留窗口圆点，让卡面更接近真实截图缩略图。
  - 六张小卡都已经从抽象占位图升级为具备明确语义的仿真截图：
    - 开发环境：VSCode + SSH
    - 推理服务：DeepSeek-V3.2 + GPU / 请求 / 负载
    - 虚拟化桌面：大号 Windows 会话窗口
    - 实验数据：多条训练折线
    - Metrics：GPU / CPU / Memory / Temp 折线
    - 数据管理：filebrowser 目录视图
- **品牌映射**：
  - 蓝色：开发
  - 紫蓝色：微调
  - 青绿色：虚拟化与推理
  - 灰蓝色：可观测性

## 入场动画序列
依赖 `.active` 类触发：
1. `0.12s`：标题区淡入。
2. `0.34s`：左侧能力列出现。
3. `0.46s`：右侧图片展示区上浮显示。
4. `0.84s`：底部结论条淡入。

## 交互设计
- 左侧能力卡和右侧展示区在 `hover` 时轻微抬升。
- 六张仿真界面保持静态产品截图感，不做复杂内部动画，避免削弱其“真实页面缩略图”观感。
