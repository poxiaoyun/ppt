# Slide 08: 一站式模型推理与微调服务

## 页面逻辑与数据映射
- **主题**：展示 Rune AI 如何把开发、微调、运行、发布与观测整合成一条可直接交付生产的模型服务链路。
- **信息映射**：
  - 顶部标题区：明确“推理与微调一体化”的主题
  - 左侧能力列：`开发 / 微调 / 虚拟化 / 弹性推理 / 全方位观测`
  - 右侧展示区：3 x 2 图片位，等待用户放入产品截图或场景图
  - 底部结论条：强调交付标准化

## 布局体系 (Capability Rail + Image Grid)
采用 **顶部标题 + 左侧五张能力卡 + 右侧 3x2 图片展示区 + 底部结论条**：
1. **顶部标题区**
   - 标题左对齐，延续章节页统一风格。
   - 副标题概括从研发到上线的完整链路。
2. **左侧五张能力卡**
   - 每张卡仅承载一个能力点，信息密度均匀。
   - 用编号与英文副标题补充结构感。
3. **右侧图片展示区**
   - 采用 3 列 2 行图片位。
   - 图片位本身不放默认图片，只提供统一占位框，便于后续替换成真实截图。
4. **底部结论条**
   - 用一句话收束“开发、微调、运行、发布、观测协同”的最终价值。

## 视觉风格
- **背景**：白底科技感，延续浅网格和中心光晕。
- **主体表现**：
  - 左侧为冷色系能力卡，右侧为整块展示面板。
  - 右侧图片位用轻边框和占位符表达“待替换图片”，避免空白突兀。
- **品牌映射**：
  - 蓝色：开发
  - 紫蓝色：微调
  - 青绿色：虚拟化与推理
  - 灰蓝色：可观测性

## 入场动画序列
依赖 `.active` 类触发：
1. `0.12s`：标题区淡入。
2. `0.34s`：左侧能力列出现。
3. `0.46s`：右侧图片展示区上浮显示。
4. `0.84s`：底部结论条淡入。

## 交互设计
- 左侧能力卡和右侧展示区在 `hover` 时轻微抬升。
- 图片位保持静态占位，不做额外动画，便于后续替换真实图片。
