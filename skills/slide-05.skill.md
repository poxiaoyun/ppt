# Slide 05: 双控制台设计

## 页面逻辑与数据映射
- **主题**：说明 Rune AI 通过双控制台设计，分别服务平台管理员与租户侧 AI 团队。
- **核心观点**：为不同角色提供专属的工作台，兼顾全局管控与敏捷开发。
- **信息映射**：
  - 左侧结构图：上方 `Console`、下方 `BOSS`，以上下关系表现“底座支撑”
  - 右侧顶部：章节标题与一句核心导语
  - 右侧主体：`BOSS 管理平台` 与 `Console 工作台` 的职责分工对照
  - 底部：一句结论，强调“双控制台”对治理与使用的平衡

## 布局体系 (Two-column + Footer Bar)
延续 **Slide 02** 的布局方法，使用 **左侧支撑关系图 + 右侧大白底说明容器 + 底部通栏结论**：
1. **顶部标题区**
   - 标题左对齐，沿用 Slide 02 的下划线装饰，保持章节切换时的统一感。
2. **左侧支撑结构**
   - 只保留两张卡：上方 `Console`，下方 `BOSS`。
   - 通过上下堆叠关系强调 `BOSS` 是 `Console` 的底座与支撑。
   - `Console` 卡内用标签形式带出 `Rune / Moha / ChatApp` 三大模块。
3. **右侧洞察容器**
   - 顶部使用小标题 `双控制台设计` 加一句核心导语，不再保留额外标签卡。
   - 中部采用紧凑的职责对照矩阵，对齐 `BOSS 管理平台` 与 `Console 工作台` 的管理对象、动作边界与输出结果。
   - 底部加入一条简洁协作说明带，直接解释“治理边界 -> 租户研发 -> 业务交付”的关系。
4. **底部结论横幅**
   - 使用高对比渐变底强调“双控制台是平衡治理与效率的关键”。

## 视觉风格
- **背景**：浅色企业科技风，与 Slide 02 保持连续性。
- **卡片风格**：左侧两张主卡与右侧双列职责卡统一采用白底、细边框、柔和阴影。
- **色彩映射**：
  - 蓝色：全局治理、平台控制
  - 绿色：租户工作区、团队协作
  - 紫色：资产协同与产品生态
  - 灰蓝色：底部协作链路与信息收束

## 入场动画序列
依赖 `.active` 类触发：
1. `0.2s`：标题自上向下淡入。
2. `0.4s`：左侧 `Console` 卡显现。
3. `0.7s`：左侧 `BOSS` 卡显现。
4. `0.6s`：右侧主容器从右向左扫入。
5. `1.0s`：底部结论横幅上移淡入。

## 交互设计
- 左侧两张主卡在 `hover` 时轻微抬升，保留与 Slide 02 一致的动效语言。
- 右侧职责对照矩阵以静态阅读为主，不再堆叠长列表，优先保证一屏扫读效率。
- 左侧通过上下结构直接表达“BOSS 支撑 Console”的层级关系，避免加入多余装饰。
