# Slide 06: 异构算力统一调度与管理

## 页面逻辑与数据映射
- **主题**：说明 Rune AI 如何把异构芯片接入、资源池化切分和智能调度整合成一套统一底座。
- **核心观点**：打破硬件壁垒，最大化算力投资回报。
- **信息映射**：
  - 顶部标题区：明确“统一调度与管理”的主题
  - 核心观点导语：一行文字收束整页，不使用独立卡片
  - 中部三段流程：`多厂商支持 -> 资源池化与切分 -> 智能调度`
  - 底部结论条：强调企业级统一算力底座

## 布局体系 (Three-Stage Pipeline Layout)
采用 **顶部标题 + 核心观点导语 + 三段式主流程 + 底部结论条**：
1. **顶部标题区**
   - 标题左对齐，延续前面章节页的统一标题语言。
   - 副标题说明异构算力统一治理带来的价值。
2. **核心观点导语**
   - 使用普通导语而非独立卡片，避免占用过多垂直空间。
3. **三段式主流程**
   - 第一段：多厂商支持，承接 GPU / NPU / DCU / PPU / XPU 等异构算力兼容，并补充本地集群、云上算力、训练 / 推理、国产替代等场景标签。
   - 第二段：资源池化与切分，采用更像产品示意图的方式，分成“共享模式 / 切分模式”两种分配路径，并落到业务价值。
   - 第三段：智能调度，承接 binpack、DRF、Gang、队列优先级等策略，并补充装箱率、公平性、分布式协同三类批调度语义。
   - 三段之间用箭头连接，强调统一链路而非孤立功能点。
4. **底部结论条**
   - 用一句话收束从“接入 -> 池化 -> 调度”的完整闭环。

## 视觉风格
- **背景**：延续白底科技感，并加入轨道、网格和中心光晕增强“底座调度”氛围。
- **主体表现**：
  - 三张主卡采用统一冷色系，但各自略有差异，体现阶段分工。
  - 当前版本已把整体信息密度重新拉平：左卡补充接入价值块，右卡补充策略说明块，中卡保留产品化共享 / 切分示意图。
- **品牌映射**：
  - 蓝色：多厂商接入与统一基础设施
  - 青绿色：资源池化与利用率提升
  - 蓝灰色：调度策略、秩序与治理
