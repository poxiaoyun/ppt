# Slide 10: Moha Hub：企业内部的 AI 资产社区

## 页面逻辑与数据映射
- **主题**：展示 Moha Hub 如何把企业内部模型、数据、镜像、应用与技能沉淀为可复用、可协作、可治理的 AI 资产社区。
- **核心观点**：Moha Hub 不只是一个仓库，而是企业 AI 资产的中枢社区，兼顾沉淀、协作、治理与对外集成。
- **信息映射**：
  - 左侧资产矩阵：`模型库 / 数据集 / 镜像仓库 / 应用空间 / MCP广场 / 模型技能`
  - 右侧主视觉：`Moha Hub Console` 产品截图主卡
  - 右侧中层：`Moha SDK CLI` 终端模拟，展示安装、登录与加密上传模型
  - 右侧亮点卡：`三层权限 / 社区协作 / 模型加解密 / Mirror + SDK`
  - 底部结论：强调私有化、资产沉淀和企业生产资料属性

## 布局体系 (Asset Matrix + Community Hub)
采用 **顶部标题 + 中部左右双区 + 底部结论条**：
1. **顶部标题区**
   - 标题直接说明“Moha Hub：企业内部的 AI 资产社区”。
   - 副标题强调私有化 AI 资产管理平台定位。
2. **摘要条**
   - 用五个轻量标签概括页面关键词：六大资产类型、三层权限、社区化协作、模型加解密、镜像站同步。
3. **左侧资产矩阵**
   - 用 2 x 3 的六张资产卡完整承接内容大纲中的“六大资产类型”，并填满整个资产面板。
   - 每张卡都包含编号、英文别名和简要说明，保持信息颗粒度一致。
4. **右侧社区中枢**
   - 右上为 `Moha Hub Console` 产品截图主卡，作为整页主视觉。
   - 截图下方插入独立的深色 `Moha SDK CLI` 模块，展示 `pip install`、`moha login` 和加密上传模型命令，强调 SDK / CLI 的可接入性。
   - 最下方四张亮点卡作为说明层，补充模型加解密、Mirror、社区化协作与 API/SDK 集成能力。
5. **底部结论条**
   - 用一句话收束 Moha Hub 对企业数据主权和资产沉淀的价值。

## 视觉风格
- **整体方向**：延续白底科技感，但相比 Slide 09 更偏“资产社区”与“协作平台”气质。
- **背景**：
  - 浅蓝网格 + 右侧绿色 / 蓝色 / 紫色柔光，形成社区中枢感。
  - 右侧增加双层轨道圆环，增强“Hub”与“社区”感觉。
- **卡片语言**：
  - 左侧六张资产卡参考 Slide 03 的 `sub-module` 风格，用不同浅色梯度区分资产类型，并统一撑满 2 x 3 矩阵。
  - 右侧截图卡比亮点卡更重，承担主视觉；中间终端模块用深底终端语言突出工程接入；右下亮点卡参考 Slide 03 的 `footer-card` 风格，更平、更轻，作为说明层。
- **品牌映射**：
  - 蓝色：模型、平台、标准化资产管理
  - 绿色：社区、协作、私有化沉淀
  - 紫色：技能、MCP、扩展生态
  - 琥珀色：能力模板和资产市场

## 入场动画序列
依赖 `.active` 类触发：
1. `0.12s`：标题区从上方淡入。
2. `0.26s`：摘要条从下方淡入。
3. `0.38s`：左侧资产矩阵整体显现。
4. `0.50s`：右侧社区中枢与亮点卡整体显现。
5. `0.82s`：底部结论条上浮入场。

## 交互设计
- 六张资产卡在 `hover` 时轻微抬升，增强“资产卡片”可浏览性。
- 右侧亮点卡在 `hover` 时也会轻微上浮，但幅度低于左侧资产卡。
- 截图主卡保持稳定，不做明显悬浮，避免影响真实产品界面的观感。

## 页面目标
- 让用户一眼看懂：Moha Hub 覆盖的资产范围不仅是模型和数据，还包括镜像、应用、MCP 和技能市场。
- 同时突出其差异化优势：私有化、权限治理、社区协作、模型加解密和对外集成能力。
