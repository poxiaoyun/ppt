# Slide 03: Rune AI 的诞生与使命

## 页面逻辑与数据映射
- **主题**：说明 Rune AI 为什么出现，以及它如何通过三大核心子系统把企业 AI 从“试点”推进到“规模化落地”。
- **核心观点**：Rune AI 是一套专为 AI 开发与部署未来而构建的企业级云原生平台，统一管理 AI 基础设施、模型与应用。
- **信息映射**：
  - 中心节点：`Rune AI` 主平台与产品定位
  - 左上模块：`Rune` 智算平台，强调推理/微调/实验/评测/应用一体化
  - 右上模块：`Moha` 魔哈仓库，强调模型、数据集、镜像等资产的沉淀与协作
  - 下方模块：`ChatApp` 模型体验，强调统一接入、治理审计、计量计费

## 布局体系 (Center Stage Layout)
采用 **白底中心发散式架构图 + 顶部使命说明 + 底部价值总结**：
1. **顶部标题区**
   - 使用 `slide-03-title`、`slide-03-subtitle` 两层结构，直接进入主题。
   - 标题区改为参考 Slide 02 的左对齐布局，并在右上角水印区域预留安全留白，避免标题与水印争抢视觉焦点。
2. **使命条 (`slide-03-mission-strip`)**
   - 以轻量的横向导语明确表达“打通底层算力、模型资产与应用入口”的使命，不再使用独立卡片容器。
   - 这是连接“市场挑战页”和“产品架构页”的桥梁。
3. **中心舞台 (`slide-03-center-stage`)**
   - 中心为高亮能量核心 `center-core`，用圆形脉冲表达“平台基座”。
   - 外围三个子模块以放射状排布，分别占据左上、右上、下方。
   - 底层使用 `SVG` 连线及流动光点表现能力协同和数据/算力流转。
4. **底部价值卡**
   - 用三张简短卡片总结“一体化 / 资产化 / 企业级”三条产品价值。

## 视觉风格
- **背景**：白色科技画布，叠加浅蓝网格、同心圆和柔和光晕，整体更明亮、更接近企业介绍页的演示场景。
- **材质**：
  - 上方三张产品卡：颜色更明确，带各自轻渐变底色和更强的产品标签感。
  - 下方三张价值卡：风格更平、更轻，作为总结信息区，不与产品卡争抢主视觉。
- **品牌色映射**：
  - 蓝色：主平台 / 基础设施 / 核心连接
  - 绿色：资产与协作
  - 紫色：模型交互、网关与治理

## 入场动画序列
依赖 `.active` 类触发：
1. `0.15s`：标题区从上方淡入。
2. `0.35s`：使命条从下方轻推入场。
3. `0.55s`：中心核心放大显现。
4. `0.95s`：连线和流动光点开始出现。
5. `1.05s / 1.20s / 1.35s`：三个子模块依次浮现。
6. `1.50s+`：底部三张价值卡错落淡入。

## 交互设计
- 子模块在 `hover` 时轻微上浮，并提高边框与阴影亮度。
- `ChatApp` 采用底部横向扁卡，避免遮挡中心核心球体，同时维持三节点放射关系。
- 页面文案与标签统一不使用 emoji，保持企业介绍场景下的专业性与一致性。
