# Slide 09: 企业级大模型网关

## 页面逻辑与数据映射
- **主题**：展示 Rune AI 如何作为企业级大模型网关，把多种调用入口、统一接入层与上游模型供给连接成一条可治理的能力链路。
- **核心观点**：网关不是单纯转发层，而是企业 AI 调用体系中的统一入口、流量编排器和治理中枢。
- **信息映射**：
  - 左侧客户端区：`AI智能体 / 具身机器人 / Vibe Coding / 智能驾驶`
  - 中央核心区：`LLM Gateway` 统一 API 入口
  - 右侧供给区：`第三方模型供应商 / Rune 平台模型服务 / 视觉、语音等小模型`
  - 底部能力卡：`统一 API 格式 / 多渠道路由与回退 / 精细化限流与计费 / 安全与合规`

## 布局体系 (Three-Stage Gateway Layout)
采用 **顶部标题 + 中部三段式链路 + 底部四张治理能力卡**：
1. **顶部标题区**
   - 标题左对齐，直接点明“企业级大模型网关”主题。
   - 副标题强调从“能调用”走向“可治理、可运营、可扩展”。
2. **中部三段式链路**
   - 左侧为调用客户端集合，右侧为上游模型接入集合，中间为 `LLM Gateway` 核心节点。
   - 左右两侧都不使用外层大卡包裹，只保留标题说明和内部信息卡，使中区更轻。
   - 中间核心球体延续 Slide 03 的中心舞台语言，作为视觉焦点。
3. **底部能力卡**
   - 四张卡横向排布，承接网关的治理能力说明。
   - 风格比中部信息卡更平，用作“能力总结层”而非主视觉。

## 视觉风格
- **背景**：白底科技画布，使用浅蓝网格、同心圆和柔和中心光晕，延续 Slide 03 的明亮科技感。
- **中部信息卡**：
  - 参考 Slide 03 的 `sub-module` 卡片语言，使用浅渐变背景、细边框和轻阴影。
  - 左侧客户端卡与右侧模型卡保持统一家族感，但通过微弱色相变化做区分。
- **底部能力卡**：
  - 参考 Slide 03 的 `footer-card` 逻辑，采用更平的总结卡样式。
  - 顶部保留短色条，作为章节内统一的结构提示。

## 数据流动效
- 页面核心动效是 **左侧 4 张卡片 -> 中间 LLM Gateway -> 右侧 3 张卡片** 的数据流动画。
- 使用 `SVG path` + `animateMotion` 实现每条通路的流动光点。
- 路径坐标不是手写死值，而是通过 [app.js](/Users/linkmaq/Project/slider-ppt/js/app.js) 根据卡片真实 DOM 位置动态计算：
  - 左侧 4 条路径分别从各客户端卡片右边缘汇聚到网关左侧
  - 右侧 3 条路径分别从网关右侧分发到各模型卡片左边缘
- 当页面内容或窗口尺寸变化时，路径会自动重算，避免连线偏离卡片。

## 入场动画序列
依赖 `.active` 类触发：
1. `0.12s`：标题区淡入。
2. `0.28s`：副标题区与整体说明进入。
3. `0.46s`：中心 `LLM Gateway` 核心球体放大显现。
4. `0.60s / 0.72s`：左右两侧信息区分别滑入。
5. `0.90s+`：`SVG` 数据流路径和流动光点开始持续运行。
6. `0.92s+`：底部四张能力卡依次淡入。

## 交互设计
- 中间左右信息卡保持较轻的 hover 反馈，不抢主视觉。
- 底部四张能力卡在 hover 时轻微上浮，增强“能力模块”层次感。
- 右侧上游模型标签统一为中性样式，不对单个供应商做颜色强调，避免视觉噪声。
